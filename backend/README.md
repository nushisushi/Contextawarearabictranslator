# Backend - Context-Aware Arabic Translator

This backend integrates dialect detection and translation for Arabic text.

## Features

- **Dialect Detection**: Classifies Arabic into EGY, LEV, GLF, IRQ, or MGH
- **Dialect Normalization**: Converts dialectal Arabic to Modern Standard Arabic (MSA)
- **Translation**: Translates normalized text to English using Helsinki-NLP/opus-mt-ar-en
- **Ambiguity Detection**: Identifies words with multiple possible meanings

## Setup Instructions

### 1. Install Dependencies

```bash
cd backend
# cd C:/Users/YOURUSERNAME/Documents/GitHub/Contextawarearabictranslator/backend
pip install -r requirements.txt
# be vigilant of errors

```
**Note**: First installation will download the Helsinki-NLP model (~300MB). This is automatic.

### 2. Prepare Training Data

You need these CSV files in the `backend/` directory:
- `dart_ready_train.csv` - Training data
- `dart_ready_test.csv` - Test data

These should have columns:
- `text` - Arabic text
- `dialect` - Dialect label (EGY, LEV, GLF, IRQ, MGH)

### 3. Train the Dialect Detector

```bash
python train_model.py
```

This will:
- Train the TF-IDF vectorizer and Logistic Regression model
- Evaluate on test set
- Save `vectorizer.pkl` and `dialect_model.pkl`
- Generate `confusion_matrix.png`

### 4. Start the API

```bash
python api.py
```

The API will start on `http://localhost:5000`

## API Endpoints

### POST /translate

Translates Arabic text with full dialect-aware processing.

**Request:**
```json
{
  "text": "حد يقدر يقول غير كدة"
}
```

**Response:**
```json
{
  "input_text": "حد يقدر يقول غير كدة",
  "detected_dialect": "Egyptian",
  "dialect_code": "EGY",
  "confidence": 0.89,
  "all_probabilities": {
    "Egyptian": 0.89,
    "Levantine": 0.06,
    "Gulf": 0.03,
    "Iraqi": 0.01,
    "Maghrebi": 0.01
  },
  "normalized_text": "حد يقدر يقول غير كده",
  "applied_normalization_rules": ["كدة → كده"],
  "translation": "Can anyone say otherwise?",
  "ambiguities": []
}
```

### GET /health

Check API status.

**Response:**
```json
{
  "status": "ok",
  "dialect_detector_loaded": true,
  "translation_model_loaded": true
}
```

### POST /test

Test with built-in example.

```
├── train_model.py             # Training script
├── requirements.txt           # Python dependencies
├── dart_ready_train.csv       # Training data (you provide)

├── vectorizer.pkl             # Generated by training
└── dialect_model.pkl          # Generated by training
```

## Models Used

1. **Dialect Detector**: Custom TF-IDF + Logistic Regression
   - Trained on DART dataset
   - Character n-grams (3-5)
   - Balanced class weights

2. **Translator**: Helsinki-NLP/opus-mt-ar-en
   - Pre-trained Arabic→English
   - MarianMT architecture
   - Auto-downloaded from Hugging Face

## Normalization Rules

The system uses dialect-specific normalization dictionaries to convert dialectal words to MSA before translation:

**Example (Egyptian)**:
- مش → ليس
- عايز → أريد  
- فين → أين

See `translator.py` for full normalization rules.

## Troubleshooting

**Models not loading:**
- Run `python train_model.py` first
- Ensure CSV files are in correct format

**Translation model download fails:**
- Check internet connection
- Verify `transformers` library is installed
- Try: `pip install --upgrade transformers`

**Low accuracy:**
- Check training data quality
- Verify CSV columns are named correctly
- Ensure balanced dialect distribution

**Out of memory:**
- Reduce `max_features` in `train_model.py`
- Use CPU instead of GPU for translation
- Reduce `max_length` in translation

## Performance

- Translation quality: Comparable to Google Translate for MSA
